В интернете компьютеры обмениваются пакетами. У каждого компьютера есть свой IP-адрес.
Чаще всего используются два протокола: UDP — быстрый, а TCP — надёжный. Каждый из этих протоколов использует порты — один компьютер может иметь много одновременных соединений.
Протокол HTTP предназначен для работы веб-сайтов. Он работает поверх TCP на порту 80.

Для анализа трафика можно использовать программу Wireshark.
Сейчас всё чаще используется HTTPS — шифрованный HTTP.

запрос типа GET происходит, когда вы переходите по ссылке или вводите адрес в адресной строке браузера, 
а POST — когда вы заполняете какую-либо форму, например авторизуетесь на сайте или отправляете сообщение.

Host	Домен из адресной строки
User-Agent	Браузер пользователя
Referer	Откуда пришёл пользователь

Запрос
GET /forensics/network.html HTTP/1.1
Accept: text/html;q=1
Accept-Language: ru-RU;q=0.5
Host: course.ugractf.ru
Referer: https://course.ugractf.ru/forensics/
User-Agent: Mozilla/5.0 (X11; Ubuntu; Linux x86_64; rv:75.0) Gecko/20100101 Firefox/75.0

Ответ
HTTP/1.1 200 OK
Content-Length: 1234
Content-Type: text/html
Date: Thu, 16 Apr 2020 13:37:00 GMT
Server: nginx

Виды ответов первой строки:
200	OK	Запрос успешен
404	Not found	Страница не найдена
403	Forbidden	Доступ запрещен
401	Unauthorized	Требуется ввести логин и пароль
302	Moved Temporarily	Переадресация
500	Internal Server Error	Ошибка сервера
